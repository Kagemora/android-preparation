# [android-preparation](https://github.com/Kagemora/android-preparation)

Статический сайт-шпаргалка для подготовки к Android-собеседованиям. Без сборки, без зависимостей — просто HTML/CSS/JS, открывается прямо из репозитория через GitHub Pages.

**Доступен по адресу:** [kagemora.github.io/android-preparation](https://kagemora.github.io/android-preparation/)

## Что внутри

**1025 карточек** вопрос-ответ и справочных материалов:

- **675 вопросов** в 18 технических категориях (Kotlin, Java, Collections, AndroidSDK, ViewXML, Compose, Concurrency, Storage, Network, Architecture, Patterns, Principles, OOP, DI, Testing, Build, Performance, Tools, KMP)
- **186 вопросов** отдельно в Algorithms и SystemDesign — задачи с кодом и решениями, включая живые задачи с реальных собеседований (не только теория)

## Формат карточки

Каждый вопрос раскрывается в четыре блока:

1. **Что дополнительно спрашивают** — связанные доп.вопросы: лёгкие сразу с кратким ответом в скобках, сложные помечены `*`
2. **Что это** — определение, с которого начинаешь отвечать
3. **Ключевое отличие** — то, что реально проверяют (сравнение/механика/причина)
4. **Пример / следствие** — где это всплывает на практике, отдельно от кода
5. **Пример кода** — рабочий сниппет, отдельным блоком с подсветкой синтаксиса

Инлайн-термины в тексте (`Handler`, `suspend`, `@Composable` и т.п.) подсвечены как пилюли-бейджи для быстрого визуального сканирования.

## Прогресс

Отметки «выучено» хранятся в `localStorage` браузера — локально, без аккаунта и синхронизации между устройствами.

## Структура файлов

```
index.html         — разметка страницы
css/styles.css      — тема (светлая, IDE-стиль)
js/categories.js    — метаданные категорий + Lifehacks-вопросы
js/data.js           — все вопросы и ответы
js/app.js            — рендер, поиск, аккордеон, прогресс, AllQuestions
```

## Как запустить локально

Просто открой `index.html` в браузере. Для гарантированного сохранения прогресса между сессиями лучше через локальный сервер, а не через `file://` напрямую — некоторые браузеры ограничивают `localStorage` на файловом протоколе.

## Как выложить на GitHub Pages

1. **Settings → Pages → Source: Deploy from a branch → Branch: main / (root)**
2. Файлы должны лежать в корне репозитория в структуре, указанной выше
3. Через 1–2 минуты сайт доступен по `https://<ник>.github.io/<репозиторий>/`
